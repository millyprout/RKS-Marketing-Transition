Core operating principle

Notion is the personal command center for knowing what needs to happen. The local desktop is the active workspace. The server is the shared source for resources and files that others need to access.

Working Standards

Start with a lightweight system that supports Helena as the primary person producing marketing work. She does not need to download the entire server or mirror every folder on her computer.

Start with an empty local workspace

Create clearly labeled local folders for active RKS and PA-AI work, but do not download the full server. Bring files onto the desktop only when they are needed for a project.

Use the desktop for active production

Keep current working files locally while writing, designing, editing, or exporting. Local work is faster and avoids repeatedly opening large production files across the server connection.

Download from the source

Before beginning a project, find the correct server folder and download the latest approved source files, templates, references, and assets needed for the task.

Download complete Creative Cloud folders

For After Effects, Illustrator, Photoshop, Premiere, and similar files, download the entire project folder. Linked images, fonts, footage, and supporting assets may not be embedded inside the main file.

Keep project assets together

Maintain the same folder structure while working locally so source files, links, images, footage, fonts, exports, and project files stay connected.

Use clear file names and versions

Include the brand, project, asset, and version in file names. Use numbered working versions and reserve “Approved” or “Final” for files that have actually completed review.

Upload when the team needs access

Upload working or review files when another person needs to see, approve, edit, or use them. Do not keep important shared materials only on the local desktop.

Return approved work to the server

When a project is complete, upload the editable source folder, approved final files, and necessary exports to the correct server location.

Avoid duplicate sources of truth

Do not create multiple competing server folders for the same project. Link Notion to the correct folder and archive obsolete versions when appropriate.

Share in accessible formats

Because the team does not use Notion consistently, share PDFs, Word files, SharePoint links, server locations, or platform drafts based on what the reviewer can access most easily.
